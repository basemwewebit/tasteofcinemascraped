# Implementation Plan: Certified Cinematic Translation Quality Engine

**Branch**: `001-translation-quality` | **Date**: 2026-03-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-translation-quality/spec.md`

---

## Summary

Add a **post-processing Translation Quality Engine** to the plugin that reviews, scores (0–100), and rewrites Arabic-translated cinematic content via the existing OpenRouter API (Claude-3.5-Sonnet). The engine is integrated as a WordPress plugin extension — two new custom DB tables, five REST endpoints, an admin review queue, and Action Scheduler for async processing of large articles. It runs automatically after every import and is also available on-demand to editors. All quality decisions are captured in an immutable audit log with role-split access (admins see all; editors see their own entries).

---

## Technical Context

**Language/Version**: PHP 8.1+ (WordPress requirement; PSR-12 strict types), Python 3.11+ (existing pipeline, no changes required)
**Primary Dependencies**: WordPress Plugin API, Action Scheduler ≥3.6 (standalone Composer package), OpenRouter API via `wp_remote_post()` (existing `OPENROUTER_API_KEY` env var reused)
**Storage**: Two new MySQL/MariaDB custom tables via `dbDelta()`: `{prefix}toc_translation_jobs`, `{prefix}toc_audit_log`; WordPress site options for configuration
**Testing**: Manual validation steps (no PHPUnit configured in project) — see Testing Checklist in [quickstart.md](quickstart.md)
**Target Platform**: WordPress ≥6.0, Linux server, same hosting environment as existing plugin
**Performance Goals**: ≤30s response for ≤2,000-word synchronous jobs; ≤5min async processing for 2,001–5,000 words
**Constraints**: Max 5,000 words per job (hard reject); 45s PHP `wp_remote_post()` timeout for sync path; Action Scheduler retry: max 3 × 10-minute intervals for async path
**Scale/Scope**: ~50–200 articles/day → ~200 jobs/day → peak 18,000 rows over 90-day retention window

## Phase 1: Design Summary

### Data Model

Two custom tables. See [data-model.md](data-model.md) for full schema.

- **`{prefix}toc_translation_jobs`** — one row per engine invocation; indexed on `post_id`, `status`, `initiating_user_id`, `created_at`
- **`{prefix}toc_audit_log`** — immutable append-only event log; indexed on `job_id`, `post_id`, `user_id`
- **`toc_quality_threshold`** site option — serialized array; `manage_options` only

**State machine**: `pending → processing → [auto-approved | flagged-for-review | engine-unavailable] → [human-approved | rejected]`

### Interface Contracts

Five REST endpoints under `/wp-json/tasteofcinemascraped/v1/quality/`. See [contracts/rest-api.md](contracts/rest-api.md).

| Endpoint | Method | Auth |
|----------|--------|------|
| `/quality/run` | POST | `edit_posts` |
| `/quality/jobs/{id}` | GET | `edit_posts` (own) / `manage_options` (any) |
| `/quality/jobs/{id}/resolve` | POST | `edit_posts` (own) / `manage_options` (any) |
| `/quality/audit` | GET | `edit_posts` (own) / `manage_options` (all) |
| `/quality/settings` | GET + PUT | `manage_options` |
