# التوثيق الشامل — tasteofcinemascraped

توثيق كامل لمشروع **tasteofcinemascraped**: سكراب مقالات Taste of Cinema، ترجمتها إلى العربية عبر OpenRouter، واستيرادها إلى ووردبريس مع الحفاظ على التاريخ والسلوق الإنجليزي وتحويل الصور إلى WebP.

---

## 1. نظرة عامة

### الهدف

- **سكراب** مقالات من [Taste of Cinema](https://www.tasteofcinema.com) (بما فيها المقالات متعددة الصفحات).
- **ترجمة** المحتوى إلى العربية مع الإبقاء على أسماء الأفلام وسنواتها بالإنجليزي.
- **استيراد** النتيجة إلى ووردبريس كمسودات، مع رفع الصور وتحويلها إلى WebP مضغوطة، وإنشاء/ربط الكتّاب والتصنيفات والوسوم.
- **الحفاظ على** تاريخ النشر الأصلي، وسلوق الروابط (المقال، الكاتب، التصنيفات، الوسوم) بالإنجليزي.
- **تصنيف رئيسي من العنوان والاكسيربت:** يُستنتج تصنيف واحد من عنوان المادة وملخصها (عبر النموذج)، يُنشأ في ووردبريس إن لم يكن موجوداً، وتُربط المادة به؛ المواد اللاحقة التي تنتمي لنفس التصنيف (نفس السلوق) تُضاف إليه دون إنشاء تصنيف جديد.
- **سكراب دفعة واحدة** لسنة كاملة أو لأرشيف مخصّص مع تخطي المقالات المستوردة مسبقاً.

### المكوّنات

| المكوّن | الوصف |
|---------|--------|
| **حزمة بايثون** `tasteofcinemascraped` | سكراب (Scrapling)، ترجمة (OpenRouter)، وإرسال الـ payload إلى ووردبريس. |
| **إضافة ووردبريس** `tasteofcinemascraped-wp` | نقطة نهاية REST للاستيراد، تحويل الصور إلى WebP، إنشاء المنشورات والوسوم والكتّاب. |

---

## 2. المتطلبات

- **Python** 3.10 أو أحدث.
- **WordPress** مع تفعيل REST API.
- **مفتاح OpenRouter** من [openrouter.ai](https://openrouter.ai) (للترجمة).
- امتداد **GD** في PHP مع دعم **WebP** (لتحويل الصور؛ إن لم يكن متوفراً تُرفع الصورة الأصلية).

---

## 3. التثبيت والإعداد

### 3.1 إضافة ووردبريس

1. ضع مجلد الإضافة في:  
   `wp-content/plugins/tasteofcinemascraped-wp/`
2. من **لوحة التحكم** → **الإضافات** فعّل **Taste of Cinema Scraped Import**.
3. اذهب إلى **الإعدادات** → **TOC Scraped**.
4. أدخل **المفتاح السري** للاستيراد (مثل `my-secret-2024`) واحفظه — ستحتاجه في ملف `.env` في جانب بايثون.

### 3.2 بيئة بايثون

من **جذر المشروع** (المجلد الذي بداخله مجلد `tasteofcinemascraped`):

```bash
cd /path/to/tasteofcinemaarabi
python3 -m venv .venv
source .venv/bin/activate   # ويندوز: .venv\Scripts\activate
pip install -r tasteofcinemascraped/requirements.txt
```

### 3.3 ملف البيئة `.env`

انسخ القالب ثم عدّل القيم:

```bash
cp tasteofcinemascraped/.env.example tasteofcinemascraped/.env
```

| المتغير | الوصف | مثال |
|---------|--------|------|
| `OPENROUTER_API_KEY` | مفتاح API من OpenRouter | `sk-or-v1-...` |
| `OPENROUTER_MODEL` | نموذج الترجمة | `openai/gpt-4o-mini` أو `anthropic/claude-3.5-sonnet` |
| `WORDPRESS_URL` | عنوان الموقع (بدون / في النهاية) | `http://tasteofcinemaarabi.test` |
| `WORDPRESS_ENDPOINT_PATH` | مسار نقطة الاستيراد | `wp-json/tasteofcinemascraped/v1/import` (افتراضي) |
| `WORDPRESS_SECRET` | نفس المفتاح السري في الإعدادات → TOC Scraped | `my-secret-2024` |

**ملاحظة:** يتم تحميل `.env` من مجلد `tasteofcinemascraped` أو من المجلد الحالي.

## 4. سير العمل (Pipeline)

```
رابط المقال (أو أرشيف/سنة)
        ↓
   [1] سكراب (Scrapling)
        ↓  استخراج: عنوان، محتوى، صور، كاتب، بايو، تاريخ، وسوم، تصنيفات
        ↓  دمج الصفحات 2، 3، … واستخراج البايو من آخر صفحة إن وُجد
        ↓
   [2] ترجمة (OpenRouter)
        ↓  ترجمة إلى العربية مع الإبقاء على أسماء الأفلام وسنواتها
        ↓  توليد سلوق إنجليزي: post_name، author_slug، slugs للوسوم والتصنيفات
        ↓
   [3] استيراد (WordPress)
        ↓  تحميل الصور → تحويل إلى WebP → رفع للميديا
        ↓  إنشاء/ربط كاتب (مع البايو)، تصنيفات، وسوم، منشور مسودة
        ↓  تخطي إن وُجد منشور بنفس source_url
```

## 5. واجهة سطر الأوامر (CLI)

يُشغَّل الأمر من **جذر المشروع** بعد تفعيل الـ venv:

```bash
cd /path/to/tasteofcinemaarabi
source .venv/bin/activate
python -m tasteofcinemascraped ...
```

### 5.1 مقالة واحدة

| الأمر | الوظيفة |
|--------|---------|
| `python -m tasteofcinemascraped "https://www.tasteofcinema.com/2026/..."` | سكراب → ترجمة → استيراد (منشور مسودة، مع الحفاظ على تاريخ النشر). |
| `python -m tasteofcinemascraped "https://..." --dry-run` | سكراب وترجمة فقط؛ لا إرسال لووردبريس (للتجربة). |
| `python -m tasteofcinemascraped "https://..." --no-translate` | سكراب واستيراد المحتوى بالإنجليزي دون ترجمة. |

### 5.2 سكراب دفعة (سنة أو أرشيف)

| الخيار | الوظيفة |
|--------|---------|
| `--year 2026` | اكتشاف كل روابط المقالات من أرشيف السنة `https://www.tasteofcinema.com/2026/` ثم معالجة كل مقالة (مع الحفاظ على تاريخ نشر كل مقال). |
| `--archive "URL"` | نفس الفكرة من رابط أرشيف مخصّص (مثلاً `/2026/` أو أي صفحة أرشيف). |
| `--delay 2` | تأخير بالثواني بين كل مقالة والأخرى (افتراضي: 2). |
| `--dry-run` | مع السنة/الأرشيف: اكتشاف + سكراب + ترجمة فقط دون استيراد. |

أمثلة:

```bash
# كل مقالات 2026
python -m tasteofcinemascraped --year 2026

# أرشيف مخصّص مع تأخير 1.5 ثانية
python -m tasteofcinemascraped --archive "https://www.tasteofcinema.com/2026/" --delay 1.5

# تجربة بدون استيراد
python -m tasteofcinemascraped --year 2026 --dry-run
```

### 5.3 ملخص الخيارات

| الخيار | القيمة | الوصف |
|--------|--------|--------|
| `url` | رابط | رابط مقالة واحدة (مطلوب إن لم يُستخدم --year أو --archive). |
| `--year` | سنة (عدد) | سكراب كل مقالات السنة من أرشيف الموقع. |
| `--archive` | رابط | سكراب كل المقالات من رابط أرشيف. |
| `--dry-run` | — | عدم إرسال لووردبريس (سكراب وترجمة فقط). |
| `--no-translate` | — | عدم ترجمة؛ إرسال المحتوى كما هو. |
| `--delay` | عدد (ثانية) | التأخير بين المقالات عند استخدام --year أو --archive (افتراضي: 2). |

## 6. واجهة REST API (الإضافة)

### 6.1 نقطة النهاية

- **الطريقة:** `POST`
- **المسار:** `{WORDPRESS_URL}/{WORDPRESS_ENDPOINT_PATH}`  
  مثال: `https://yoursite.com/wp-json/tasteofcinemascraped/v1/import`

### 6.2 المصادقة

يجب تمرير المفتاح السري بأحد الشكلين:

- **رأس الطلب:**  
  `X-Tasteofcinema-Secret: your-secret`
- **أو معامل في الجسم/استعلام:**  
  `secret=your-secret`

إن لم يُضبط المفتاح في الإعدادات أو كان غير صحيح، تُرجع نقطة النهاية خطأ (503 أو 403).

### 6.3 جسم الطلب (JSON)

| الحقل | النوع | إلزامي | الوصف |
|--------|--------|--------|--------|
| `source_url` | string | نعم | الرابط الأصلي للمقال (يُستخدم لمنع الاستيراد المزدوج). |
| `title` | string | نعم | عنوان المنشور. |
| `content` | string | نعم | محتوى المنشور (HTML). |
| `post_name` | string | لا | سلوق المنشور (إنجليزي)؛ إن وُجد يُستخدم للرابط الدائم. |
| `author_name` | string | نعم | اسم الكاتب (للعرض). |
| `author_bio` | string | لا | وصف الكاتب (يُحفظ في حقل الوصف). |
| `author_slug` | string | لا | سلوق/لوجن الكاتب (إنجليزي). |
| `date` | string | لا | تاريخ النشر الأصلي (قابل للتحليل عبر `strtotime`). |
| `tags` | array | لا | وسوم: إما نصوص أو `[{ "name": "...", "slug": "..." }]`. |
| `categories` | array | لا | تصنيفات: نفس شكل الوسوم. |
| `images` | array | لا | مصفوفة عناصر: `{ "url": "...", "alt": "..." }` أو مجرد روابط. |

### 6.4 الاستجابة

- **201:** استيراد ناجح — `{ "success": true, "post_id": 123 }`.
- **200 مع تخطي:** المقال مستورد مسبقاً — `{ "skipped": true, "message": "...", "post_id": 123 }`.
- **403/503:** خطأ مصادقة أو إعدادات.
- **500:** خطأ من الخادم — `{ "error": "رسالة الخطأ" }`.

## 7. معالجة الصور (WebP)

- بعد تحميل كل صورة من الرابط الأصلي، تحوّلها الإضافة إلى **WebP** بجودة مضغوطة (ثابت `TCOC_SCRAPED_WEBP_QUALITY` = 82).
- التحويل يتم عبر **GD** في PHP (`imagecreatefromjpeg/png/gif/webp` ثم `imagewebp`).
- إن فشل التحويل (مثلاً عدم دعم WebP أو ملف تالف)، يُرفع الملف الأصلي دون تحويل.
- الروابط في المحتوى تُستبدل بروابط الملفات المرفوعة في مكتبة الميديا؛ أول صورة تُستخدم كصورة مميزة للمنشور.

## 8. السلوق والبايو وتاريخ النشر

### 8.1 السلوق بالإنجليزي

- **منشور:** إن وُجد `post_name` (مولَّد من العنوان الإنجليزي) يُستخدم كلقب للمنشور في ووردبريس.
- **كاتب:** إن وُجد `author_slug` يُستخدم كلوجن للمستخدم؛ وإلا يُستنتج من الاسم.
- **وسوم وتصنيفات:** إن أُرسلت بصيغة `{ "name": "الاسم بالعربي", "slug": "english-slug" }` يُستخدم الـ slug الإنجليزي عند الإنشاء.

### 8.2 البايو (وصف الكاتب)

- يُستخرج من نهاية المحتوى (كتلة "Author Bio") ولا يُدرج في جسم المنشور.
- إن وُجد البايو في الصفحة الأخيرة فقط (مثلاً صفحة 2)، يُؤخذ من هناك.
- يُحفظ في ووردبريس في حقل **الوصف** للكاتب (user description).

### 8.3 تاريخ النشر

- الحقل `date` المُرسل من بايثون (من السكراب) يُمرَّر إلى `post_date` في ووردبريس، فيُحافظ على تاريخ النشر الأصلي للمقال.

### 8.4 التصنيف الرئيسي (من العنوان والاكسيربت)

- بعد ترجمة العنوان والمحتوى، يُستنتج **تصنيف واحد** من العنوان والاكسيربت (أول ~400 حرف من النص) عبر استدعاء النموذج.
- النموذج يُرجع سطرين: اسم التصنيف بالعربية، وسلوق إنجليزي (مثلاً `crime-thriller-movies`). يُطلب منه استخدام **نفس السلوق** لنفس نوع المواضيع لدمج المواد تحت تصنيف واحد.
- هذا التصنيف يُضاف في رأس قائمة التصنيفات المُرسلة لووردبريس. الإضافة تنشئ التصنيف إن لم يكن موجوداً (حسب السلوق)، أو تربط المنشور بالتصنيف الموجود إن وُجد نفس السلوق — فلا يُنشأ تصنيف جديد للمواد اللاحقة التي تنتمي لنفس التصنيف.

## 9. السكراب الدفعي (أرشيف / سنة)

- الدالة `discover_article_urls_from_archive(archive_url)` تفتح رابط الأرشيف (مثلاً `/2026/`) وتتتبع التصفحة (`/page/2/`, …) حتى لا تُضاف روابط جديدة.
- تُجمَع فقط الروابط التي تُعتبر مقالات (نفس النطاق، نفس بادئة المسار، دون روابط التصفحة أو التعليقات).
- الروابط المُكررة (نفس المقال مع/بدون `/2/`) تُوحَّد إلى رابط المقال الأساسي.
- بعد جمع القائمة، تُنفَّذ للمقالة الواحدة نفس الخطوات: سكراب → ترجمة → استيراد؛ المقالات التي سبق استيرادها (نفس `source_url`) تُتخطى.

## 10. هيكل الحزمة (بايثون)

| الملف | الوظيفة |
|--------|----------|
| `__main__.py` | نقطة الدخول لسطر الأوامر؛ تنسيق الخيارات وتشغيل السكراب/الترجمة/الإرسال. |
| `config.py` | قراءة المتغيرات من البيئة وبناء رابط استيراد ووردبريس. |

| `wordpress_client.py` | إرسال الـ payload إلى نقطة استيراد ووردبريس. |

| الثابت | القيمة | الوصف |
|--------|--------|--------|
| `TCOC_SCRAPED_OPTION_SECRET` | `tasteofcinemascraped_secret` | مفتاح خيار المفتاح السري في ووردبريس. |

## 12. استكشاف الأخطاء ونصائح

- **تشغيل من المجلد الخطأ:** يجب تشغيل `python -m tasteofcinemascraped` من **جذر المشروع** (المجلد الذي بداخله مجلد `tasteofcinemascraped`)، وليس من داخل `tasteofcinemascraped` نفسه.
- **OPENROUTER_API_KEY غير مضبوط:** مطلوب للترجمة؛ مع `--dry-run` إن لم يُضبط يتم تخطي الترجمة وطباعة ملخص.
- **WORDPRESS_SECRET غير مطابق:** المفتاح في `.env` يجب أن يكون مطابقاً تماماً للمفتاح في **الإعدادات → TOC Scraped**.
- **الصور لا تتحول لـ WebP:** تأكد من دعم PHP لـ GD و `imagewebp`؛ إن لم يكن متوفراً تُرفع الصورة بصيغتها الأصلية.
- **مقالات مكررة:** الاستيراد يتخطى أي منشور له نفس `source_url` (يُخزَّن في post meta). لا حاجة لحذف يدوي للمكررات من واجهة الاستيراد.
- **تأخير عند الدفعة:** استخدم `--delay` لتقليل الضغط على خادم المصدر وووردبريس عند سكراب سنة أو أرشيف كبير.
